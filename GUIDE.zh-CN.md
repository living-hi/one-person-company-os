# One Person Company OS 通俗指南

把这个技能理解成“一人公司的可视化经营驾驶舱”最准确。

它从一个可卖承诺、一个第一批买家、一个当前瓶颈和一个今日最短动作开始，把产品、销售、交付、回款、学习和可复用资产放在同一条经营闭环里推进。

## 一句话理解

经营闭环是：

```text
价值承诺 -> 买家 -> 产品能力 -> 交付 -> 回款 -> 学习 -> 资产
```

每次认真运行，都应该让这条闭环更清楚：

- 承诺清楚到可以对外销售
- 买家具象到可以直接触达
- 产品能力窄到可以交付
- 交付和回款可见
- 反馈能沉淀成学习
- 重复工作能变成资产

## 你怎么用

第一次最简单：

```text
我正在创建一个 AI 一人公司，请调用 one-person-company-os v1.0。先帮我确认可卖承诺、第一批买家和核心问题；确认后在我批准的本地目录创建工作区，用经营驾驶舱作为入口，直观展示当前瓶颈，并且只把批准后的文件保存到这个工作区内。
```

系统应该先确认：

- 公司和产品名称
- 可卖承诺
- 第一批买家
- 核心问题
- 当前主目标
- 当前主瓶颈
- 当前主战场：销售、产品、交付、回款或资产
- 今日最短动作
- 创始人批准的工作区路径

只有创始人确认工作区后，才真正创建文件。

## 第一个打开什么

中文工作区从这里开始：

```text
阅读版/00-经营驾驶舱.html
```

驾驶舱会链接到可编辑工作面和正式交付件。先从这里判断下一步打开什么，不需要在文件夹里盲找。

## 工作区三层

- HTML 页面用于下载后的第一阅读和导航。
- Markdown 文件是持续编辑的工作底稿。
- `产物/` 下的编号 DOCX 是正式交付件。
- `视觉素材/` 下的 SVG 承载精确经营闭环和成交管道图。
- AI 图片提示词是可选创作素材，用于封面和发布帖。

## 一次好的运行应该结束于

- 当前瓶颈可见
- 当前负责角色或主战场清楚
- 今日最短动作明确
- 保存路径清楚，或者明确说明未保存原因
- 状态变化后同步刷新阅读层、Markdown 工作层和视觉素材
- 价格、法务、预算、发布和高风险客户动作先请求创始人确认

## 你应该记住的 6 个动作

- 说清痛点
- 找到第一批买家
- 收窄可卖承诺
- 交付最小可用能力
- 把交付连到回款
- 把重复工作沉淀成资产
