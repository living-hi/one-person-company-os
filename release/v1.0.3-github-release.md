# v1.0.3: Mainstream Platform Push List

`one-person-company-os` v1.0.3 publishes the completed mainstream marketplace push package.

This release keeps the core visual operating cockpit runtime unchanged. The update is about distribution quality: every mainstream channel now has a clear local package, status, blocker, next action, and evidence rule.

## Included Channels

- OpenClaw / ClawHub
- Claude Skills
- OpenAI GPT Store
- MCP Registry, Smithery, Glama, and PulseMCP
- Hermes Agent / agentskills.io
- Dify Marketplace
- GitHub Copilot Extensions
- Microsoft Copilot Studio / Commercial Marketplace
- Poe Bots
- Gemini Gems

## What Improved

- `platforms/PUBLISHING-STATUS.md` is now the single canonical push table.
- MCP registry submission notes were added for the official registry, Smithery, Glama, and PulseMCP.
- agentskills.io listing copy was added for the Hermes-compatible distribution.
- Microsoft Commercial Marketplace notes now separate tenant agent publication from Partner Center marketplace submission.
- OpenAI GPT Actions notes now include the public privacy-policy requirement.
- GitHub Copilot guidance now avoids the retired GitHub App Copilot Extension route and treats the currently supported Copilot path as `backend-needed`.
- Platform validation fails if mainstream submission materials are missing.

## Publication Boundary

OpenClaw / ClawHub can be verified by API and downloadable package. Other channels remain `ready-to-submit`, `backend-needed`, or `tenant-needed` until their platform account, hosted backend, tenant, approval, or public listing evidence exists.

Do not mark a platform as `published` without a listing URL, version API, downloadable package, or review record.
