# Workflow: Build To Launch

## Goal

Take an approved MVP or release from scope to launch readiness with controlled quality and clear distribution planning.

## Step 1: Lock Scope

Confirm:
- the release objective
- in-scope features
- out-of-scope items
- release risks

Artifacts:
- PRD
- sprint plan

## Step 2: Define Technical Delivery

Establish:
- architecture choices
- implementation milestones
- dependencies
- observability requirements

Artifacts:
- architecture note
- implementation checklist

## Step 3: Build

Execute:
- feature development
- integration work
- environment setup
- operational preparation

Artifacts:
- code changes
- release notes draft

## Step 4: Test And Harden

Check:
- critical user flows
- edge cases that matter for release
- performance or reliability risks
- rollback readiness

Artifacts:
- test plan
- bug list
- release readiness summary

## Step 5: Prepare Launch Motion

Create:
- launch narrative
- landing page updates
- onboarding assets
- support and FAQ materials

Artifacts:
- launch brief
- launch checklist
- sales or support scripts

## Step 6: Final Approval

Require explicit founder approval before:
- production deployment
- mass outreach
- pricing changes
- public claims about capability or compliance

## Exit Criteria

The workflow is complete when:
- release quality is acceptable for the stage
- launch assets are ready
- monitoring and rollback thinking exist
- owner and next actions after launch are explicit
