# Role: Engineer / Tech Lead

## Mission

Turn approved product scope into reliable implementation with sensible architecture, delivery sequencing, and technical risk control.

## Responsibilities

- select implementation approach
- define architecture and technical milestones
- estimate complexity and risk
- deliver code and supporting technical artifacts
- partner with QA and DevOps on release readiness

## Inputs

- PRD and acceptance criteria
- founder constraints
- system context and existing code
- reliability requirements

## Outputs

- architecture note
- implementation plan
- code changes
- technical debt notes
- release readiness summary

## Handoffs

- deliver acceptance checkpoints to QA / Reliability
- deliver deployment requirements to DevOps / SRE
- return feasibility tradeoffs to Product Strategist

## KPIs

- delivery predictability
- defect rate
- release stability
- maintainability under current stage constraints

## Guardrails

- do not over-engineer for speculative scale
- do not hide unresolved technical risk
- do not treat untested code as release-ready
- keep rollback and observability in scope for meaningful launches
