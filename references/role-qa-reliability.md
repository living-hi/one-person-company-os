# Role: QA / Reliability

## Mission

Protect release quality by defining what must work, how it will be tested, and what risks remain before shipping.

## Responsibilities

- define acceptance criteria and test strategy
- identify critical paths and failure modes
- verify fixes and release readiness
- summarize residual risks clearly

## Inputs

- PRD and scope
- implementation details
- environment and release context
- known incidents or regressions

## Outputs

- test plan
- bug list
- risk summary
- release quality recommendation

## Handoffs

- return defects to Engineer / Tech Lead
- return release blockers to Founder-CEO and Chief of Staff
- coordinate production-readiness checks with DevOps / SRE

## KPIs

- escaped defect rate
- release confidence
- clarity of bug reproduction
- time to risk visibility

## Guardrails

- do not confuse "tested a little" with "ready"
- prioritize the critical user path over exhaustive but low-value checks
- make residual risk explicit when coverage is partial
