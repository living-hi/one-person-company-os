# Agent Runtime Guide

本仓库优先服务 OpenClaw。

## v1.0 运行原则

- 用户始终是创始人和最终决策者。
- 智能体系统默认跟随用户语言：中文用户默认中文，英文用户默认英文。
- 首次进入必须先确认可卖承诺、第一批买家和核心问题，再落地工作区。
- 系统以经营闭环推进，不再以旧阶段/回合推进。
- 用户默认应先看到经营驾驶舱：主目标、主瓶颈、主战场、成交管道、经营闭环和今日最短动作。
- 角色默认最小激活，避免第一次就铺满全公司角色。

## 默认智能体

首次确认后，默认只创建：

- `创始人`
- `总控台`
- `产品负责人`

按需再创建：

- `设计负责人`
- `增长负责人`
- `用户运营`
- `质量保障`
- `数据分析`
- `财务`
- `法务合规`
- `运维保障`

## v1.0 交接上下文

所有角色交接都必须保留这些字段：

- 当前主目标
- 当前主瓶颈
- 当前主战场
- 价值承诺
- 第一批买家
- 产品状态
- 成交管道
- 交付与回款状态
- 今日最短动作
- 是否需要创始人确认

## 推荐执行顺序

1. 确认可卖承诺、第一批买家和核心问题
2. 等创始人确认保存目录
3. 创建 v1.0 经营工作区
4. 生成经营驾驶舱、阅读层和视觉素材
5. 只推进一个当前主瓶颈
6. 每次真实推进后刷新 markdown、HTML 和视觉素材

## 不该做的事

- 不要谎称已经创建了智能体
- 不要在未确认时批量写文件
- 不要把文件写到用户未确认的工作区之外
- 不要默认进入长周期周报模式
- 不要重新引入旧阶段/回合状态
- 不要让 AI 图片承载精确经营数字；精确数据图使用确定性 SVG/HTML
